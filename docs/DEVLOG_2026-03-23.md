# 開発ログ 2026-03-23

## 方針整理

`docs/GPTlog` の内容に合わせ、プロジェクトの主軸は「音声反応ツール」ではなく「GaussianSplatting データのビューワー兼プレイグラウンド」と整理した。

音声反応は主役ではなく、Splat に追加できる演出レイヤーとして扱う。

## フェーズ 1: MVP 土台

- Vite + React + TypeScript の初期構成を作成
- Three.js / R3F / postprocessing を導入
- procedural な背景演出を実装
- 音声ファイルの読込と周波数解析を実装

## フェーズ 2: Gaussian Splat 最小統合

- `@mkkellogg/gaussian-splats-3d` を導入
- `DropInViewer` を Three scene に統合
- `.ply` / `.splat` / `.ksplat` / `.spz` のローカル読込を追加
- Splat status 表示を追加

## フェーズ 3: UI 改善への追従

- 改修済み UI に合わせて機能を再配置
- メトリクスバー、カスタムスライダー、ネオントーンの UI に追従

## フェーズ 4: Splat transform と Scene export / import

- `x / y / z / rotationY / baseScale` を追加
- 数値入力 UI を追加
- scene export / import を追加
- `splatFileName` を使った remap 待ちを追加

## フェーズ 5: gizmo と remap 操作

- `TransformControls` を追加
- `translate / rotate / scale` を切替可能にした
- gizmo enable / disable を追加
- import 後の pending remap に対して適用 / 破棄操作を追加

## フェーズ 6: 複数 Splat 対応

今回の更新で以下を実施。

- 単一 Splat 前提の state を廃止
- 複数 Splat を `splats[]` として保持
- Splat outliner を追加
- 選択中 Splat を切り替え可能にした
- 数値 transform 編集を選択対象ベースに変更
- gizmo を選択中 Splat のみに適用
- scene export / import を複数 Splat 対応に更新
- pending remap を複数ファイル対応に更新

## 現在の成果

- 複数 Splat を同時に読み込める
- outliner から対象を切り替えて編集できる
- scene 設定を複数 Splat 付きで保存 / 読込できる
- 音声反応を補助演出として重ねられる

- pointer-based placement なし
- 画像 / 動画 / glTF 未統合
- scene graph / layer 管理なし
- bundle が大きい
